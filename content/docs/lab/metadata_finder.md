# MetaDataFinder
MetaDataFinder is a local search tool for exploring metadata files generated by MetaDataMaker.
It helps you quickly locate specific experiments by scanning a selected folder for valid MDM `.json` files, indexing their metadata fields, and letting you filter the resulting files through structured queries.
The search runs locally in the browser, so the selected metadata files are inspected on your computer without being uploaded anywhere.

## Quickstart guide
Start by clicking `Search MDM Files`, then select the target folder containing your MDM metadata files.
The browser will recursively scan the selected folder and its subfolders for `.json` files, but only files whose parsed JSON contains the top-level field `"generated_by": "MetaDataMaker"` are kept.

After the scan, all valid MDM files are listed in the upper results window, together with basic statistics such as the number of files found.
The tool also inspects those files to collect all available metadata keys and sub-keys, which are then made available in the _Key_ dropdown menu.

Build your filter query by adding `Key-Value` rows and connecting them with logical operators: `AND`, `OR`, and `XOR`.
Use `(` and `)` to define priorities explicitly.
Buttons are enabled or disabled automatically to help prevent invalid query structures.

Each `Key-Value` row lets you choose a metadata key, a comparison operator, and a value.
The available comparison operators are `=`, `≠`, `>`, `<`, `≥`, and `≤`.
The `=` operator performs the standard matching behavior; an empty value with `=` means "_key exists_".
The `≠` operator means "_key exists, but its value does not match the specified value_".

For string values, matching is __case-insensitive__ and __substring-based__.
For booleans and numbers, the code first tries strict boolean or numeric matching, then falls back to string matching.
The comparison operators `>`, `<`, `≥`, and `≤` are applied only to pure numeric values or strict ISO dates in the format `YYYY-MM-DD`.

The query is parsed using Boolean precedence: `AND` before `XOR`, `XOR` before `OR`, while parentheses override precedence.
Use `Apply Filter` to show the filtered results in the bottom results window.
